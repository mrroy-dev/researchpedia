# The Reading Room

A Vite + React site for browsing and reading your research paper collection —
styled like a library card catalog, with an in-app PDF reader.

## Pages

- `/` — landing page with a hero search, "pick up where you left off",
  "trending on this device", and "recently catalogued" shelves
- `/browse` — the full catalog: search, A-Z tab filter, pagination
- `/paper/:id` — the reader (metadata sidebar + embedded PDF)

Every route change uses a book-page-turn animation.

### About "trending" — read this

This is a static site with no server or database, so there's no way to see
what *other* people are reading. "Trending on this device" and "pick up
where you left off" are tracked in `localStorage`, scoped to one browser.
It's a real, useful feature for you personally, but it is **not** shared
across visitors. If you want genuine multi-user trending (everyone's reads
counted together), that needs a small backend — e.g. a serverless function
+ a database (Supabase, Postgres, etc.) that records a view whenever
`/paper/:id` loads. Happy to help build that if you want it.

## Run it

```bash
npm install
npm run dev
```

Open the printed localhost URL. It currently shows 18 sample papers so you can
see the design before adding your own.

## Add your ~1000 papers

1. Copy your PDF files into `public/papers/` (any filenames, e.g.
   `attention-is-all-you-need.pdf`).
2. Run:
   ```bash
   npm run scan
   ```
   This reads every PDF in `public/papers/`, pulls the title/author out of
   each file's embedded metadata (falling back to a title made from the
   filename when a PDF has none), and regenerates `src/data/papers.json`.
3. Restart `npm run dev` (or just refresh — Vite hot-reloads JSON changes).

Re-run `npm run scan` any time you add more PDFs; it fully regenerates the
catalog from whatever is currently in `public/papers/`.

### Cleaning up metadata

The scanner does its best, but PDF metadata is often missing or messy. Open
`src/data/papers.json` afterward and fix up titles, authors, year, and add
`tags` (used for the search/subjects) wherever you want cleaner results —
it's a plain JSON array, one object per paper.

## Build for deployment

```bash
npm run build
```

Outputs a static site to `dist/` — deploy it anywhere (Vercel, Netlify,
GitHub Pages, a plain server). Because the reader loads PDFs straight from
`public/papers/`, just make sure that folder ships along with `dist/`.

## Structure

- `src/pages/Library.jsx` — the catalog grid, search, and A-Z tab filter
- `src/pages/Reader.jsx` — in-app reader (metadata sidebar + embedded PDF)
- `src/data/papers.json` — the catalog data (regenerated by `npm run scan`)
- `scripts/scan-papers.mjs` — the metadata scanner
- `public/papers/` — put your PDF files here
